# Reflexión final del proyecto

## Introducción

Durante el desarrollo de Stardew Planner he podido trabajar en un proyecto fullstack completo, combinando frontend con React, TypeScript y Tailwind CSS, y backend con Node.js, Express y TypeScript.

Este proyecto me ha permitido aplicar de forma práctica muchos de los conceptos vistos durante la formación, no solo a nivel técnico, sino también en organización, documentación y conexión entre distintas partes de una aplicación real.

## Qué he aprendido

Uno de los principales aprendizajes de este proyecto ha sido entender mejor cómo se conecta un frontend moderno con un backend propio mediante una API REST.

También he reforzado conocimientos sobre:

- creación de componentes reutilizables en React
- uso de TypeScript para tipar datos, props y respuestas de la API
- gestión de estado local y global
- uso de hooks de React
- organización de un backend por capas
- validación básica de datos en el servidor
- estructura y documentación de un proyecto fullstack

Además, el proyecto me ha ayudado a entender mejor que una aplicación no consiste solo en hacer que algo se vea bien, sino en organizar bien el código, mantener coherencia entre frontend y backend y documentar correctamente las decisiones tomadas.

## Problemas encontrados durante el desarrollo

Durante el desarrollo aparecieron varios problemas, sobre todo relacionados con la integración entre frontend y backend y con algunos detalles de tipado en TypeScript.

Entre los principales problemas encontrados destacan:

- errores en imports y configuración inicial del proyecto
- ajustes necesarios en tipos compartidos entre cliente y servidor
- problemas con valores posiblemente nulos al trabajar con varias fuentes de datos
- errores de tipado en parámetros de rutas del backend
- necesidad de coordinar correctamente estado local, contexto global y peticiones a la API
- pequeños fallos en operaciones CRUD hasta dejar la lógica estable

Estos problemas fueron resolviéndose paso a paso, revisando la estructura del código y corrigiendo cada parte antes de seguir avanzando.

## Uso de IA durante el desarrollo

Durante el desarrollo del proyecto he utilizado inteligencia artificial como apoyo para resolver dudas, revisar errores, proponer estructuras de código y redactar parte de la documentación técnica.

La IA me ha servido especialmente para:

- organizar mejor el proyecto desde el inicio
- resolver errores de configuración y tipado
- plantear una estructura clara para frontend y backend
- redactar documentación de forma más ordenada
- revisar decisiones técnicas durante el desarrollo

Aun así, no se ha tratado de copiar soluciones sin más, sino de utilizar la IA como herramienta de apoyo, entendiendo y adaptando cada parte al proyecto concreto.

## Conclusión final

En conjunto, Stardew Planner ha sido un proyecto útil para poner en práctica el desarrollo de una aplicación fullstack completa, desde la idea inicial hasta la estructura del frontend, el backend, la API, la interacción del usuario, la documentación y el despliegue.

El proyecto me ha permitido afianzar conceptos técnicos importantes y entender mejor cómo se construye una aplicación de forma organizada, modular y progresiva.

Además, me ha servido para comprobar que la planificación, la documentación y la revisión continua son tan importantes como el propio código.
